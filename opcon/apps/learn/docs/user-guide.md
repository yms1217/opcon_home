## 1. 앱 개요

로봇 데이터학습플랫폼은 **학습 데이터 수집부터 모델 배포 승인까지**의 전체 흐름을 하나의 화면에서 관리합니다.

```
Task 선택  →  수집 방법 선택  →  데이터 수집  →  NAS 저장  →  Forge 전달  →  학습 실행  →  검증/승인  →  배포
                (5가지 방법)                  (데이터 센터)  (학습 플랫폼)
```

모든 raw data는 먼저 **NAS 서버(데이터 센터 내)**에 저장된 후, Forge(학습 플랫폼)로 전달됩니다.

### 학습 데이터 수집 방법 5가지

| 방법 | 설명 | 권장 용도 |
|---|---|---|
| **TMS 실행** | Taskflow를 실행하고 그 데이터를 학습에 활용 | 운영 중인 Task의 지속적 개선 |
| **Teleoperation** | 사람이 직접 로봇을 조종하여 고품질 데이터 수집 | 정밀 조작, 새로운 Task 학습 |
| **Learn-by-Watching** | 사람 작업 영상에서 모션 데이터 추출 | 빠른 초기 학습, 단순 Task |
| **시뮬레이션/증강** | 기존 데이터를 증폭하거나 합성 데이터 생성 | 데이터 부족 보완, Edge case |
| **기존 데이터 업로드** | 이미 확보된 Dataset을 직접 업로드 | 외부 데이터, 재활용 데이터 |

## 2. 메인 화면 — 학습 시작

**경로**: `/learning/`  
**사이드바**: 학습 시작

### 단계 1 — Task 선택

화면 진입 시 가장 먼저 **어떤 Task를 학습시킬지** 선택합니다.

1. TMS에 등록된 Task 목록이 카드 형태로 표시됩니다.
2. 학습할 Task 카드를 클릭하여 선택합니다 (선택 시 파란 테두리로 강조).
3. 목록에 없는 신규 Task라면 하단 **새 Task 직접 입력** 필드에 이름을 입력합니다.
4. **다음 →** 버튼을 클릭합니다.

> 선택한 Task는 이후 모든 방법 페이지(Teleop, LbW, Upload 등)에서 자동으로 적용됩니다.

### 단계 2 — 학습 방식 선택

선택한 Task가 상단에 표시되고, 5가지 수집 방법 카드가 나타납니다.

1. 원하는 수집 방법의 카드를 클릭합니다.
2. 해당 방법의 상세 설정 화면으로 이동합니다.

- **← Task 변경** 버튼으로 Task 선택 단계로 돌아갈 수 있습니다.

> **팁**: 어떤 방법이 내 상황에 맞는지 모를 경우 [학습 방법 선택 가이드](#12-학습-방법-선택-가이드)를 참고하세요.

## 3. TMS로 학습 데이터 수집

**경로**: `/learning/tms`  
**사이드바**: 학습 시작 > TMS 학습

### 개요

기존에 운용 중인 Taskflow를 실행하면서 그 실행 데이터를 학습용으로 수집합니다. 별도로 데이터를 만들지 않아도, 로봇이 실제 작업하는 과정 자체가 학습 데이터가 됩니다.

### Step 1 — Taskflow 선택

1. 목록에서 학습에 활용할 Taskflow를 선택합니다.
2. 선택한 Taskflow의 설명과 Step 수를 확인합니다.
3. **다음** 버튼을 클릭합니다.

> Taskflow가 목록에 없다면 TMS 앱에서 먼저 Taskflow를 등록하세요.

### Step 2 — 실행 설정

| 설정 항목 | 설명 |
|---|---|
| 수행 로봇 | 실행에 투입할 로봇을 선택합니다 (복수 선택 가능) |
| 반복 횟수 | 동일 Taskflow를 몇 번 반복 실행할지 설정합니다 |
| 실행 목적 | 데이터 수집 / 성능 검증 / 실패 케이스 수집 중 선택 |
| 학습용 저장 | ON으로 설정 시 실행 결과가 Episode 후보함에 저장됩니다 |

> **주의**: 로봇을 최소 1대 이상 선택해야 다음 단계로 진행됩니다.

### Step 3 — 확인 및 실행

설정 내용을 최종 확인하고 **🚀 실행 시작** 버튼을 클릭합니다.

### Step 4 — 실행 결과 모니터링

- 실행 상태가 자동으로 갱신됩니다 (3초 간격).
- 진행률 바로 전체 진행 상황을 확인합니다.
- 실행 완료 후 **Episode 후보함으로 이동 →** 버튼을 클릭하면 [Episode 후보 검토](#4-episode-후보-검토) 화면으로 이동합니다.

## 4. Episode 후보 검토

**경로**: `/learning/tms/episodes/:executionId`

### 개요

TMS 실행에서 생성된 Episode 하나하나를 검토하고, 학습에 사용할 것을 선별합니다. 선별된 Episode만 Forge Dataset으로 전송됩니다.

### 화면 구성

화면은 좌우로 나뉩니다.
- **왼쪽**: Episode 목록 (필터 + 스크롤 리스트)
- **오른쪽**: 선택한 Episode의 상세 정보 및 검토 패널

### 필터 사용법

| 필터 | 설명 |
|---|---|
| Step | 특정 Step에서 발생한 Episode만 표시 |
| 상태 | 성공 / 실패 / 재시도 필터링 |
| Intervention | 사람이 개입한 Episode만 표시 |

### Episode 검토

Episode를 클릭하면 오른쪽 패널에 상세 정보가 표시됩니다.

| 버튼 | 동작 |
|---|---|
| **채택** | 이 Episode를 학습 데이터로 사용합니다 |
| **보류** | 나중에 다시 검토합니다 |
| **제외** | 이 Episode는 학습에 사용하지 않습니다 |

### NAS 저장 및 Forge 전달

화면 하단의 요약 바에서 채택 수를 확인하고 2단계로 진행합니다.

1. **NAS에 저장 (N개 Episode)** 버튼 클릭 → 채택된 Episode가 NAS 서버에 저장됩니다.
2. NAS 저장 완료 메시지 확인 후 **Forge로 전달** 버튼 클릭 → Forge 학습 플랫폼으로 데이터가 전달됩니다.
3. 완료 후 **Forge에서 보기 →** 버튼으로 Forge Dataset을 확인합니다.

> **팁**: 실패 Episode도 학습에 유용할 수 있습니다. 실패 패턴 학습(Failure Recovery)을 위해 의도적으로 실패 Episode를 채택하는 것을 검토해보세요.

## 5. Teleoperation으로 정밀 시연 수집

**경로**: `/learning/teleop`  
**사이드바**: 학습 시작 > 텔레오퍼레이션

### 개요

사람이 직접 로봇을 원격 조종하며 고품질의 action 데이터를 수집합니다. Forge Data Collector가 실제 데이터 수집을 담당합니다.

### 사용 방법

1. **Task 선택**: 수행할 Task를 드롭다운에서 선택합니다.
2. **목표 Episode 수**: 수집할 목표 수량을 입력합니다.
3. **로봇 선택**: 조종할 로봇을 드롭다운에서 선택합니다.
4. **학습 목적**: Fine-tuning / Failure Recovery / Benchmark Dataset 생성 중 선택합니다.
5. **🎮 Forge Data Collector 열기** 버튼을 클릭합니다.

> Task와 로봇을 선택해야 버튼이 활성화됩니다.

Forge Data Collector가 새 탭에서 열리며, 등록된 세션 ID로 데이터 수집이 시작됩니다.

## 6. Learn-by-Watching으로 영상에서 시작

**경로**: `/learning/watch`  
**사이드바**: 학습 시작 > 영상 학습

### 개요

사람이 작업하는 영상을 업로드하면 Forge가 자동으로 모션 데이터를 추출합니다. 로봇 없이도 학습 데이터를 빠르게 확보할 수 있습니다.

### Step 1 — 영상 유형 선택

| 유형 | 설명 |
|---|---|
| 1인칭 영상 (Ego View) | 작업자 시점에서 촬영된 영상 |
| 3인칭 영상 (Exo View) | 외부 카메라로 촬영된 영상 |
| 혼합 | 두 가지 유형을 함께 사용 |

### Step 2 — 활용 목적 선택

- 사전학습 (Pre-Training)
- 간단 Task 학습
- 모션 추출 / Retargeting
- 증강용 Seed 데이터 생성

복수 선택이 가능합니다.

### Step 3 — 품질 기대치 확인

이 방법의 장점과 한계가 자동으로 표시됩니다. 반드시 읽고 다음 단계로 진행하세요.

### Step 4 — 영상 업로드 및 Forge 전송

1. **영상 파일 업로드**: 드래그 & 드롭 또는 클릭으로 파일을 추가합니다.
   - 지원 포맷: MP4, MOV, AVI
2. **최소 메타데이터 입력**: Task명, 영상 수, 포맷 등을 입력합니다.
3. **NAS에 저장** 버튼을 클릭합니다. → NAS 저장 완료 메시지 확인
4. **Forge로 전달 및 모션 추출 시작** 버튼을 클릭합니다.

전달 완료 후 Forge Video to Motion 화면으로 바로 이동할 수 있습니다.

## 7. 시뮬레이션/증강으로 데이터 확대

**경로**: `/learning/simulation`  
**사이드바**: 학습 시작 > 시뮬레이션/증강

### 개요

기존 데이터를 기반으로 다양한 변형 데이터를 생성하거나, 합성 데이터를 만들어 데이터셋을 확장합니다.

### Step 1 — 입력 데이터 유형 선택

| 유형 | 설명 |
|---|---|
| 기존 데이터셋 기반 증강 | 이미 확보한 실제 데이터를 변형하여 확대 |
| 이미지 기반 Synthetic Video | 이미지에서 합성 영상을 생성 |
| 비디오 기반 Motion 생성 | 영상에서 모션 데이터를 추출 |

### Step 2 — 목표 설정

- 데이터 수량 확대
- Edge case 생성
- 환경 다양화
- Scene Rebuilding

### Step 3 — Forge 기능 연결

목적에 맞는 Forge 도구를 선택하여 실행합니다.

| 도구 | 설명 |
|---|---|
| Mimic Augmentation | 실제 데이터를 변형하여 다양한 학습 데이터 생성 |
| WFM Synthetic Data | World Foundation Model로 합성 데이터 생성 |
| Video to Motion | 영상에서 모션 데이터를 추출하고 변환 |

## 8. 기존 데이터 업로드

**경로**: `/learning/upload`  
**사이드바**: 학습 시작 > 데이터 업로드

### 개요

이미 확보한 Dataset 파일을 직접 업로드하여 Forge 학습에 연결합니다.

### 사용 방법

1. **데이터 유형 선택**: 업로드할 데이터의 유형을 선택합니다.
   - Teleop dataset
   - Fleet/TMS 로그 dataset
   - Human video dataset
   - 외부 benchmark / 공개 dataset

2. **메타데이터 입력**:

| 항목 | 설명 |
|---|---|
| Task명 (필수) | 이 데이터가 대응하는 Task 이름 |
| Robot type | RSP-7 / RSP-9 선택 |
| 모달리티 | Vision / State / Force / Vision+State |
| 성공/실패 라벨 유무 | 라벨이 있는지 여부 |
| 학습 용도 | Pre-Training / Fine-tuning / Benchmark |

3. **파일 업로드**: 드래그 & 드롭 또는 클릭으로 파일을 추가합니다.
   - 지원 포맷: JSON, HDF5, ZIP

4. 포맷 검사 결과를 확인합니다. ⚠️ 표시가 나오면 포맷을 확인하세요.

5. **NAS에 저장** 버튼을 클릭합니다. → NAS 저장 완료 메시지 확인
6. **Forge로 전달** 버튼을 클릭합니다. → 학습 플랫폼으로 전달 완료

> **주의**: 데이터 유형과 파일이 모두 선택되어야 저장 버튼이 활성화됩니다.

## 9. 데이터 현황 대시보드

**경로**: `/learning/data`  
**사이드바**: 데이터 현황

### 개요

현재까지 수집된 학습 데이터의 전체 현황을 3가지 관점에서 확인합니다.

### 탭 1 — 데이터 출처별 현황

각 수집 방법(TMS, Teleop, LbW, Simulation, Upload)별로 수집된 Episode/영상 수와 검토 상태를 보여줍니다.

### 탭 2 — 학습 목적별 준비 상태

Pre-Training / Post-Training / In-Field / Failure Model 등 학습 목적별로 데이터가 얼마나 준비되었는지 진행률로 표시합니다.

### 탭 3 — Task별 준비 상태

자제 팔레타이징, Pick & Place 등 개별 Task별로 현재 보유 Episode 수와 준비 상태를 확인합니다.

## 10. 학습 실행 현황

**경로**: `/learning/training`  
**사이드바**: 학습 현황

### 개요

Forge에서 실행 중인 모든 학습 Job의 상태를 모니터링합니다.

### 화면 구성

각 학습 Job 카드에서 다음을 확인할 수 있습니다.

| 정보 | 설명 |
|---|---|
| Job 이름 | 학습 Job의 식별 이름 |
| 기반 모델 | 사용 중인 Foundation Model (OpenVLA-OFT, π0, RDT-1B 등) |
| Dataset | 학습에 사용되는 Dataset ID |
| 시작 시간 / 경과 시간 | 학습 시작 시각과 현재까지 걸린 시간 |
| 진행률 | 실행 중인 Job의 진행 퍼센트 |
| 상태 배지 | 대기중 / 실행중 / 완료 / 실패 |

- **Forge에서 보기 →** 버튼으로 Forge Training 상세 화면으로 이동합니다.
- **새로고침** 버튼으로 최신 상태를 다시 불러옵니다.

## 11. 검증 / 승인

**경로**: `/learning/review`  
**사이드바**: 검증/승인

### 개요

학습이 완료된 모델의 Validation 결과를 확인하고, 실제 배포 여부를 결정합니다.

### 모델 카드 구성

각 모델 카드에서 다음을 확인하고 조치를 취합니다.

- **기반 모델**: 어떤 Foundation Model 기반인지
- **생성일**: 학습 완료 시각
- **Validation Score**: 검증 점수 (진행률 바 + 퍼센트)
  - 🟢 85% 이상: 우수
  - 🟡 70~85%: 보통
  - 🔴 70% 미만: 미달

### 조치 버튼

| 버튼 | 동작 | 설명 |
|---|---|---|
| ✓ 승인 | 모델을 배포 승인 | 이미 승인된 모델은 비활성화됨 |
| ✕ 반려 | 모델을 배포 반려 | 이유를 기록하는 것을 권장 |
| ↩ 재학습 요청 | 해당 모델을 재학습 대기 상태로 변경 | 데이터 보완 후 재학습 |
| Forge Simulation 보기 | Forge에서 시뮬레이션 결과 상세 확인 | |

## 12. 학습 방법 선택 가이드

상황에 따라 적합한 방법이 다릅니다.

### 시나리오별 권장 방법

**"지금 운용 중인 Task를 더 잘하게 하고 싶다"**
→ **TMS 실행** — 실제 운용 데이터를 바탕으로 지속적인 개선

**"완전히 새로운 Task를 로봇에게 가르치고 싶다"**
→ **Teleoperation** — 가장 고품질의 데이터, 처음부터 정확히 시연

**"빨리 시작해보고 싶다, 영상 자료는 있다"**
→ **Learn-by-Watching** — 영상으로 빠른 초기 학습, 단 정밀 조작에는 한계

**"데이터가 부족하다, 특수 상황 대비가 필요하다"**
→ **시뮬레이션/증강** — 기존 데이터를 기반으로 다양한 케이스 생성

**"과거에 수집한 데이터가 있다"**
→ **기존 데이터 업로드** — 이미 확보된 데이터를 즉시 활용

### 학습 단계별 권장 방법

| 학습 단계 | 권장 방법 |
|---|---|
| Pre-Training (사전학습) | Learn-by-Watching, 시뮬레이션/증강, 기존 데이터 업로드 |
| Post-Training (파인튜닝) | TMS 실행, Teleoperation |
| In-Field (현장 지속 개선) | TMS 실행 (운용 중 자동 수집) |
| Failure Recovery | TMS 실행 (실패 케이스 목적), Teleoperation |
